# software-requirements-hangman

### Project Objective

The objective of this project is to perform requirements elicitation for an online hangman game, it focuses on the software engineering process rather than on implementing the hangman game itself. It aims to analyze the problem domain and define clear software requirements, emphasizing requirements elicitation, structured documentation, version control with Git, and organized publication practices.

### Requirements Document

The requirements are documented in the file Requisitos_Forca.md, located in the root directory of this repository. The file can be viewed directly through the GitLab interface or by cloning the repository locally and opening the document in any Markdown-compatible editor.
